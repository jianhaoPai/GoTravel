# 旅伴地图 MVP

一个原生微信小程序 MVP，用来让朋友在同一个旅行房间里收藏打卡点、查看彼此想去的地点，并把地点手动排序成路线后打开地图导航。

## 已实现

- 创建旅行房间：填写名称、城市、日期后创建。
- 微信分享加入：从分享卡片进入时读取 `tripId`，并调用加入房间逻辑。
- 共享地图：同一旅行里的收藏点展示在地图上，用收藏人标签区分。
- 地点收藏：通过微信地图选点，补充分类和备注。
- 筛选查看：按成员、分类查看收藏点。
- 地点详情：展示收藏人、备注、想去人数、评论列表，并支持“我也想去”和导航。
- 路线规划：选择候选地点，手动拖动或用上下按钮排序，保存路线并打开地图导航。
- CloudBase 封装：页面统一调用 `services/api.js`，没有配置云环境时自动使用 mock 数据。

## 运行方式

1. 用微信开发者工具导入当前目录。
2. 如果只是看原型，保留 `project.config.json` 里的 `touristappid` 即可。
3. 如果要接入真实云开发：
   - 把 `project.config.json` 的 `appid` 换成你的小程序 AppID。
   - 现在云环境 ID 集中放在 `config/env.js`，把 `cloudEnvId` 改成你的云开发环境 ID。
   - 在微信开发者工具里上传并部署 `cloudfunctions` 下的云函数。
   - 创建数据库集合：`trips`、`members`、`places`、`comments`、`placeReactions`、`routePlans`。
   - 使用云函数读写数据；`database.rules.json` 默认关闭客户端直连读写，成员权限由云函数校验。

更详细步骤见 `CLOUD_SETUP.md`。


## 关键文件

- `pages/home`：旅行首页、创建房间、成员/分类筛选、地点列表。
- `pages/map`：共享地图、地点标记、添加地点。
- `pages/route`：路线候选、手动排序、保存路线、打开导航。
- `pages/place`：地点详情、想去、评论展示、单点导航。
- `services/api.js`：云函数调用与本地 mock 兜底。
- `cloudfunctions/*`：CloudBase 云函数实现。
